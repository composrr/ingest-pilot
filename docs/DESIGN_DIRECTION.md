# Design Direction: Ingest Pilot

The app should feel like a modern native Mac utility: calm, rounded, polished, and fast. It should pair the visual comfort of a contemporary productivity app with the clear, practical workflow of Post Haste.

## Visual Principles

- Warm off-white app background, not stark white.
- Left sidebar navigation with compact icon + label items.
- Large rounded main content panel with subtle border and soft shadow.
- Rounded controls, but keep dense workflow screens efficient.
- Black primary buttons for decisive actions like Start Ingest or Save Preset.
- Muted neutral surfaces with a small lavender accent for identity and selection.
- Avoid a marketing-site feel. This is a working desktop tool.

## Product Feel

- Presets should feel like reusable workflows, not abstract configuration files.
- The main screen should make the next ingest obvious.
- Preset authoring should feel friendlier than Post Haste while preserving its speed.
- Technical safety details should be visible when useful, quiet when not.

## Compact Preset Authoring

Post Haste is visually plain, but its density is useful. Ingest Pilot should borrow that efficiency without copying the styling.

- Preset setup should fit as much as possible in one working view.
- Prefer compact tables for variables, routing, and setup values.
- Prefer a dense folder tree over large cards when editing templates.
- Keep row heights tight, around 32-40px, with subtle borders.
- Use tabs or segmented controls to switch between Setup, Folders, Routing, and Template Files.
- Keep authoring controls closer to desktop utility software than web dashboard cards.
- Use rounded panels and warm surfaces from the brand, but reduce padding inside work surfaces.
- Provide previews inline in a footer/status strip instead of large explanatory areas.
- Preserve keyboard-driven speed: table rows, quick add, duplicate, delete, and inline editing.
